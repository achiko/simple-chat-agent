     Context

     Running the stack (docker compose up -d) and submitting a prompt today
     produces no usable error signal in the UI. Two concrete failures were
     observed:

     1. A stale NextAuth JWT whose userId doesn't exist in User triggers a
     Postgres FK violation inside POST /api/sessions. The unhandled rejection
     crashes the Next.js process (ELIFECYCLE exit 143), so the browser only ever
      sees a generic HTTP 500.
     2. Once past that, the worker throws AI_LoadAPIKeyError: OpenAI API key is 
     missing. The chat message eventually renders as "No output generated. Check
      the stream for errors." ‚ÄĒ a dead-end message that forces a trip to docker 
     compose logs.

     Both failures are technically already visible to the server: the worker
     writes the message string to Job.error, and routes return { error: "..." }
     JSON. But the shape is inconsistent, unguarded DB errors bypass it
     entirely, and the client (components/chat-tab/index.tsx:168, 339, 410)
     flattens everything to a bare toast.error(string). SSE stream errors
     (es.onerror at line 369) and SWR fetch errors on the History/Gallery/System
      tabs are silent.

     Goal: every failure surfaces in the UI as a clear, actionable message ‚ÄĒ a
     title, a one-line hint, and an optional "Details" disclosure ‚ÄĒ without the
     user needing to read docker logs. Keep it proportional to this app's size;
     no Sentry, no telemetry backend.

     // lib/api-errors.ts
     export type ErrorPayload = {
       code: string;       // machine tag, e.g. "auth.stale_session"
       message: string;    // one human sentence ready to show to the user
       hint?: string;      // short actionable next step
       detail?: string;    // collapsed "Details" text (server-side message / 
     constraint name)
     };

     export class AppError extends Error {
       constructor(
         public readonly status: number,
         public readonly payload: ErrorPayload,
         cause?: unknown,
       ) { super(payload.message, { cause }); }
     }

     // Known codes (centralized for client + server):
     export const ERRORS = {
       validation:        (detail: string) => new AppError(400, { code:
     "validation", message: "Your input wasn't accepted.", hint: "Check the 
     highlighted fields and try again.", detail }),
       unauthorized:      ()              => new AppError(401, { code:
     "auth.unauthorized", message: "You're signed out.", hint: "Refresh the page
      to sign in as a guest." }),
       staleSession:      ()              => new AppError(409, { code:
     "auth.stale_session", message: "Your session is out of date.", hint:
     "Refresh the page to start a new guest session." }),
       forbidden:         ()              => new AppError(403, { code:
     "auth.forbidden", message: "You don't have access to this resource." }),
       notFound:  (what: string)          => new AppError(404, { code:
     "not_found", message: `${what} wasn't found.` }),
       aiKeyMissing:      ()              => new AppError(500, { code:
     "ai.key_missing", message: "The AI provider isn't configured.", hint: "Set 
     OPENAI_API_KEY in .env.local and restart the worker." }),
       streamDisconnected:()              => new AppError(0,   { code:
     "stream.disconnected", message: "Lost connection to the stream.", hint:
     "Retry to reconnect." }),
       internal:   (detail?: string)      => new AppError(500, { code:
     "internal", message: "Something went wrong on the server.", hint: "Try 
     again in a moment.", detail }),
     };

     mapToAppError(err: unknown): AppError ‚ÄĒ central translator:
     - AppError ‚Üí return as-is.
     - ZodError ‚Üí ERRORS.validation(issues[0].message).
     - Postgres error with code === "23503" and constraint_name containing
     "User_id_fk" ‚Üí ERRORS.staleSession().
     - AI_LoadAPIKeyError (from AI SDK) ‚Üí ERRORS.aiKeyMissing().
     - Anything else ‚Üí ERRORS.internal(err.message) (and console.error the full
     cause server-side).

     withErrorHandler(fn) ‚ÄĒ route wrapper that calls fn(req, ctx), catches,
     calls mapToAppError, returns NextResponse.json({ error: payload }, { status
      }). For auth.stale_session, also sets Set-Cookie: authjs.session-token=; 
     Max-Age=0; Path=/ and __Secure-authjs.session-token=; ‚Ä¶ so the browser
     drops the orphaned JWT and the next request re-runs the guest flow.
